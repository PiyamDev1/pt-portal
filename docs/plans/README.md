# Planning Documents

This folder holds planning material that should be kept in the repository but should not sit at the repo root.

## What lives here

- codebase cleanup and refactoring plans
- historical maintenance audits
- feature planning documents
- long-form implementation handoff notes

## Subfolders

- `future/` for feature and product planning

## Notes

- These files are intentionally retained for context.
- They are planning artifacts, not the primary source of truth for current implementation details.
- Current developer-facing documentation lives under `docs/guides/` and `docs/technical/`.
